## ğŸ¯ Key Architectural Decisions

### 1. Virtual Module Scheme
**Decision:** Use `\0figma-asset:` virtual module prefix  
**Reason:** Vite convention for custom loaders, avoids file path conflicts  
**Benefit:** Clean separation between dev and production resolvers

### 2. Extension Flexibility
**Decision:** Auto-convert between `.png` and `.jpg`  
**Reason:** Figma exports can be inconsistent, developers shouldn't worry  
**Benefit:** Robust against format changes, no code updates needed

### 3. Fail-Safe Build
**Decision:** Build fails with clear errors if corruption detected  
**Reason:** Broken images on production are worse than failed builds  
**Benefit:** Issues caught early, clear fix instructions provided

### 4. Multiple Validation Layers
**Decision:** 4 independent layers of protection  
**Reason:** Defense in depth, catch issues at multiple points  
**Benefit:** Very low chance of corrupted images reaching production

- **Magic Bytes Reference:**
  - JPEG: `FF D8 FF` (255 216 255 in decimal)
  - PNG: `89 50 4E 47` (137 80 78 71 in decimal)
  - Base64 JPEG starts with: `/9j/`
  - Base64 PNG starts with: `iVBORw0KGgo`

- **Extension Priority:**
  - Exact match first (if code imports `.png`, try `.png` first)
  - Fallback to alternate extension (try `.jpg` if `.png` missing)
  - Fail only if neither exists

- **Build Performance:**
  - Extension auto-conversion adds minimal overhead (~1-2ms per file)
  - Corruption detection only reads first 10 bytes (fast)
  - Total impact: negligible on build time
